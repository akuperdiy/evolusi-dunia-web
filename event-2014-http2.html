2014: HTTP/2

← Kembali ke Timeline

⚙️ Evolusi Besar Protokol Web

  • Standar: IETF (RFC 7540)
  • Fitur kunci: Multiplexing, HPACK, server push, prioritization

Pengertian

HTTP/2 adalah pembaruan besar protokol HTTP yang memungkinkan banyak permintaan/respons berbagi satu koneksi (multiplexing) dengan overhead lebih kecil melalui kompresi header.

Selain itu, server dapat mengirim resource proaktif (server push) dan mengatur prioritas, sehingga latensi keseluruhan berkurang dibanding HTTP/1.1.

HTTP/2 meningkatkan efisiensi transport data antara klien dan server. Dengan multiplexing, beberapa permintaan berjalan dalam satu koneksi, mengurangi latensi dan hambatan head-of-line pada HTTP/1.1.

HTTP/2 mewarisi banyak konsep dari SPDY (Google) dan mengoptimalkan performa tanpa memerlukan perubahan besar di layer aplikasi.

Fitur Teknis

  • Stream multiplexing: Banyak request/respons dalam satu koneksi TCP.
  • HPACK: Kompresi header untuk mengurangi overhead.
  • Server push: Server dapat mengirim resource proaktif.
  • Prioritization: Mengatur prioritas pengiriman resource.

Dampak bagi Pengembang

  • Mengurangi kebutuhan teknik bundling berlebihan demi mengurangi request.
  • Praktik optimasi bergeser ke strategi push/prioritas yang tepat.

Miskonsepsi Umum

  • "HTTP/2 selalu lebih cepat": Manfaat tergantung kondisi jaringan/server; konfigurasi buruk dapat mengurangi keuntungan.
  • "Server push selalu baik": Penggunaan tidak tepat bisa membuang bandwidth.
